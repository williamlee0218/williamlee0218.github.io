修改ssh远程服务器端口及ss端口、密码

发表于 2019-03-31

下午还在愉快的用着SS，结果到了晚上突然Google主页打不开了，顿时吓了一跳，ss端口被封了？没办法先下个Lantern(蓝灯),至少这个能打开搬瓦工的主页(实名吐槽搬瓦工的主页还得翻墙才能打开，真em…….)

打开搬瓦工的主页进入你的KiwiVM面板，然后打开连接https://kiwivm.64clouds.com/main-exec.php?mode=blacklistcheck，点击Test，若出现IP BLOCK, 恭喜你中奖了；若是出现IP NOT BLOCKED,那么你还能再抢救一下。

赶紧测试一下，还好没有被封，要不然就要花钱换IP地址了。

接下来测试一下KiwiVM面板的端口有没有被封，使用Xshell软件或者Putty，看你装了哪个，直接输入命令：

若是正常应该是直接可以连接，若是不行则显示： connection failed。显然我的情况属于后者。

好吧，只能改端口了。使用KiwiVM的Interactive Shell，用户名是root，密码不显示输入，手打，ok，顺利进入。其实这个Interactive Shell有些类似Xshell直接连接服务器，接下来输入: vi /etc/ssh/sshd_config。 因为我的服务器是CentOS7，若是用vim直接创建了一个新的路径，所以还是乖乖用vi命令。拉到最下面，port: xxx。insert, 手动修改一个新的端口，esc后输入：wq保存。

对了，记得在服务器防火墙上开放这个你修改的端口，要不然依旧无法连接。

若是你的ss端口失效，那么修改方式为：

  1. 远程连接服务器

  2. 输入 vi /etc/shadowsocks.json

  3. 修改端口

  4. 修改完成后：ESC，再输入:wq保存退出

  5. 重启ss即可。

    P.S. 启动ss服务代码：

    启动：service shadowsocks start
    停止：service shadowsocks stop
    重启：service shadowsocks restart
